## 二、用户与权限表

### `roles` — 角色表

| 字段名            | 类型               | 可空 | 默认值                                          | 说明                                                         |
| ----------------- | ------------------ | ---- | ----------------------------------------------- | ------------------------------------------------------------ |
| `id`              | `bigint unsigned`  | ❌    | `AUTO_INCREMENT`                                | 主键                                                         |
| `name`            | `varchar(50)`      | ❌    | —                                               | 角色名称（学校自定义）                                       |
| `code`            | `varchar(50)`      | ❌    | —                                               | 角色编码（系统生成，唯一）                                   |
| `description`     | `varchar(255)`     | ✅    | —                                               | 角色描述                                                     |
| `level`           | `tinyint unsigned` | ❌    | `0`                                             | 角色层级：0=系统 1=学生 2=教师 3=辅导员 4=系主任 5=院长 6=校长 7+=自定义 |
| `is_system`       | `tinyint unsigned` | ❌    | `0`                                             | 0=学校自定义 1=系统内置（不可删除）                          |
| `is_auditor`      | `tinyint unsigned` | ❌    | `0`                                             | 0=非审批角色 1=可作为审批节点                                |
| `scope_types`     | `json`             | ✅    | `null`                                          | 允许绑定的范围类型：\[2,3,4] 表示可绑学院/专业/班级          |
| `max_scope_count` | `int unsigned`     | ❌    | `0`                                             | 最大绑定数量：0=无限制                                       |
| `status`          | `tinyint unsigned` | ❌    | `1`                                             | 0=禁用 1=启用                                                |
| `created_at`      | `timestamp`        | ❌    | `CURRENT_TIMESTAMP`                             | 创建时间                                                     |
| `updated_at`      | `timestamp`        | ❌    | `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间                                                     |
| `deleted_at`      | `timestamp`        | ✅    | —                                               | 软删除时间                                                   |

**索引：** `UNIQUE(code)`, `INDEX(level)`, `INDEX(is_system, status)`, `INDEX(is_auditor, status)`

**系统内置角色（is_system=1, 不可删除）：**

| code              | name       | level | is_auditor | scope_types | 说明         |
| :---------------- | :--------- | :---- | :--------- | :---------- | :----------- |
| `super_admin`     | 超级管理员 | 0     | 0          | null        | 系统最高权限 |
| `admin`           | 管理员     | 0     | 0          | null        | 系统管理     |
| `student`         | 学生       | 1     | 0          | null        | 普通学生     |
| `teacher`         | 科任教师   | 2     | 0          | [4]         | 任课教师     |
| `counselor`       | 辅导员     | 3     | 1          | [4]         | 班级管理     |
| `department_head` | 系主任     | 4     | 1          | [3]         | 专业管理     |
| `college_dean`    | 院长       | 5     | 1          | [2]         | 学院管理     |
| `principal`       | 校长       | 6     | 1          | [1]         | 全校管理     |
| `auditor`         | 审核员     | 2     | 1          | [1,2,3,4,5] | 独立审核员（V8新增） |

**学校自定义角色示例（is_system=0）：**

| code              | name       | level | is_auditor | scope_types | 说明                         |
| :---------------- | :--------- | :---- | :--------- | :---------- | :--------------------------- |

| `lab_director`    | 实验室主任 | 8     | 1          | [2,3]       | 某学校自定义，可审批实验项目 |

> **多态关联数据清理策略（新增）**
>
> 系统中存在多处多态关联（`xxx_type` + `xxx_id`），包括 `audit_logs`、`system_logs`、`messages`、`pending_approvals`、`user_favorites`、`file_uploads` 等。
>
> **风险：** 当主表记录被物理删除或软删除后，多态关联表中的 `xxx_type`/`xxx_id` 指向的记录可能已不存在，形成孤儿数据，导致日志表、文件表数据膨胀，存储浪费。
>
> **清理方案：**
> 1. **应用层级联删除**：业务代码中删除主表记录（如 `archives`、`award_applications`）时，同步删除关联的多态记录。软删除场景下，同步将关联记录标记为 `deleted_at`。
> 2. **定时任务兜底**：每日凌晨运行清理任务，扫描并删除 `related_type` / `related_id` 或 `auditable_type` / `auditable_id` 指向已不存在的孤儿数据。
> 3. **归档策略**：日志类表（`system_logs`、`audit_logs`）按时间分区或归档，避免长期积累。
> 4. **软删除引用失效**：对于 `file_uploads` 等被其他表外键引用的多态表，软删除后需确保引用表（如 `career_plans.file_id`）同步更新为 NULL 或失效，避免前端展示已删除文件导致 404。

### `teacher_class_relations` — 教师-班级-课程关系视图（V8新增）

**类型：** 视图（VIEW），基于 `teacher_courses` 表联表查询

**用途：** 方便查询教师与班级、课程、专业、学院之间的关联关系，直接展示教师姓名、班级名称、专业学院层级、学期等已解析信息。

**定义 SQL：**
```sql
CREATE OR REPLACE VIEW teacher_class_relations AS
SELECT
    tc.id               AS relation_id,
    tc.teacher_id       AS teacher_id,
    u.name              AS teacher_name,
    u.user_no           AS teacher_no,
    tc.class_id         AS class_id,
    c.name              AS class_name,
    c.grade             AS class_grade,
    m.id                AS major_id,
    m.name              AS major_name,
    col.id              AS college_id,
    col.name            AS college_name,
    tc.course_code      AS course_code,
    tc.course_name      AS course_name,
    tc.semester_id      AS semester_id,
    s.name              AS semester_name,
    tc.is_primary       AS is_primary,
    tc.status           AS status,
    tc.school_id        AS school_id
FROM teacher_courses tc
JOIN users u ON u.id = tc.teacher_id AND u.deleted_at IS NULL
JOIN classes c ON c.id = tc.class_id AND c.deleted_at IS NULL
JOIN majors m ON m.id = c.major_id AND m.deleted_at IS NULL
JOIN colleges col ON col.id = m.college_id AND col.deleted_at IS NULL
JOIN semesters s ON s.id = tc.semester_id AND s.deleted_at IS NULL;
```
